## 1. 프로젝트 개요

- **프로젝트명**: 거상 드랍 아이템 감지 프로그램
- **개발 기간**: 2026.05.25 ~ 2026.05.26
- **개발환경/기술스택**:
    - Python
    - Scapy, Npcap
    - Tkinter GUI
    - Wireshark 기반 패킷 분석
    - CSV 기반 아이템 DB 매핑

---

## 2. 주요 기능

### (1) 실시간 드랍 감지

- 거상 서버 패킷 실시간 캡처
- 특정 조건의 드랍 패킷 탐지 (해당 조건으로 특정 패킷 확인)
- 특정 Opcode 기반 드랍 이벤트 판별
- 서버(`211.???.??.??:????`) → 클라이언트 방향 패킷만 필터링

---

### (2) 아이템 이름 매핑

- 패킷 내부 `item_id` 추출
- `item_db.csv`와 자동 매칭
- 드랍된 아이템 이름 실시간 출력
- 존재하지 않는 아이템은 `Unknown(item_id)` 형태로 출력

예시:

```
십전대보탕
낡은장궁
```

---

### (3) 중복 감지 방지

- 동일 패킷 재수신 시 중복 출력 방지
- Payload Hash 기반 중복 제거 로직 구현
- 일정 시간(TTL) 이후 캐시 자동 정리

---

### (4) GUI 응용프로그램

- Tkinter 기반 Windows GUI 구현
- Start / Stop 버튼 제공
- 실시간 드랍 로그 출력
- 특정 아이템 필터 검색 기능 구현

예시:

```
[18:32:11]
드랍 감지 | 십전대보탕 | item_id=110 | 수량=1
```

---

## 3. 차별화된 기술 포인트

- **Npcap + Scapy 기반 패킷 스니핑**
    
    → WinPcap 호환 모드 기반 실시간 TCP 패킷 캡처 구현
    
- **Wireshark 기반 프로토콜 역분석**
    
    → 반복적인 Hex Dump 비교를 통해 드랍 패킷 구조 직접 분석
    
- **패킷 구조 추론 및 데이터 파싱**
    
    → Opcode, Payload Offset, Item ID 위치를 역으로 추론하여 구현
    
- **CSV 기반 아이템 DB 시스템**
    
    → item_id ↔ item_name 자동 매핑 구조 구현
    
- **GUI 기반 실시간 모니터링**
    
    → 콘솔이 아닌 Windows 응용프로그램 형태로 구현
    
- **중복 감지 제거 로직**
    
    → TCP 재전송/중복 수신 문제 해결
    

---

## 4. 패킷 분석 과정 및 구현 내용

### (1) 드랍 패킷 특정

- Wireshark를 활용해 반복적으로 드랍 상황 비교
- 다음 공통 패턴 확인:

```
예)
tcp.len == ??
Opcode == 7? 2? 0? 0?
```

- 드랍 발생 시 동일 구조 패킷 반복 확인

---

### (2) 아이템 데이터 구조 분석

패킷 내부 구조 예시:

```
예)
59 00 00
77 27 00 00
...
0d 00 00 00
...
6e 00 00 00
```

- 특정 Offset에서 item_id 추출
- 드랍 아이템마다 반복되는 고정값 비교 분석

---

### (3) 아이템 DB 매핑 구현

- 거상 트레이더스 내부 DB 구조 분석
- CSV 기반 item_id ↔ item_name 매핑 구현

예시:

```
110 → 십전대보탕
607 → 낡은장궁
```

---
